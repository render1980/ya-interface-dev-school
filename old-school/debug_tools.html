Приемы отладки

Можно выделить два основных типа: прерывающие поток выполнения и, сответственно, не прерывающие поток.

Приемы, прерывающие поток выполнения:

  • Вызов alert()
  • Установка breakpoint или conditional breakpoint
  • Использование инструкции 'debugger' в скрипте

Вызов alert()

Не лучший вариант для целей отладки. Появляется модальное окно с заданным содержимым, при этом тормозя выполнение скрипта.

Установка breakpoint или conditional breakpoint

Данный метод, как и любой, заключающийся в остановке потока, плох тем, что при его использовании данные асинхронных вызовов могут быть утеряны.

Для установки conditional breakpoint необходимо задать условие, при котором скрипт будет останавливаться на этой точке.

Использование 'debugger'

Данная инструкция позволяет задать точку прерывания в скрипте.

Использование Console

Ну и, наконец, приемы, позволяющие избежать прерывания потока, заключаются в использовании console.log. Появившись на свет вместе с Firebug, данный объект сейчас существует для всех основных браузеров. Схема использования аналогична предыдущим методам: нужно лишь в требуемом месте вызвать console.log('...') с параметром, после чего сообщение появиться в отладочной консоли браузера. Кроме того, console предоставляет расширенные средства для отображения сообщений (форматирование, группировка, профилирование). Подробно об этом можно посмотреть здесь.

Сценарии отладки

Собственно, отладка может идти по двум сценариям:

  1. Браузер показал ошибку в консоли => перейти по ссылке.
  2. Браузер ошибку не показал => локализовать область возникновения ошибки => отлаживать вышеописанными способами.
