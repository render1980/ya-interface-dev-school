Об изучении

Задание: расскажите, в каком объёме и при каких обстоятельствах вы изучали следующие темы: языки программирования (императивные/декларативные, разные системы типов), структуры (классы, структуры данных, примитивные типы), парадигмы (события, ООП, параллельность/асинхронность, кеширование), алгоритмы (парсинг, сортировка, поиск), паттерны.

Языки программирования

Декларативные

С языками данного типа мне довелось ознакомиться в рамках учебной программы в университете.

Prolog рассматривался в дисциплине "Системы искусственного интеллекта". Были изучены понятия логики предикатов, основных структурных единиц построения отношений: предложения, термы, атомы, числа, переменные, структуры. Также решены несколько базовых задач: наибольший общий делитель, сортировка.

С другим декларативным языком (если его можно отнести к данному типу), Haskell, я ознакомился самостоятельно. Получил представление о базовых конструкциях языка: переменные, типы, классы, операторы, функции высшего порядка, функторы, лямбда-выражения.

Императивные

Из компилируемых это C, C++, C#, Java.

C, C++ изучались в течение двух лет в дисциплинах Системное Программирование, Информационные Технологии и в ряде подобных. Данные языки использовались (как и java) при выполнении курсовых работ. C# использовался при выполнении выпускной дипломной работы. Далее перечислю основные изученные темы для конкретного языка.

C

  • Основные конструкции языка: типы, операторы, функции, заголовочные файлы, директивы препроцессора, модули, поточный ввод/вывод, структуры
  • Указатели
  • Программирование на C в Linux
  • Обзор архиитектуры операционных систем Windows, Unix с применением языка C

С++

Т.к. данный язык является надмножеством C, то все базовые конструкции имеют место быть и здесь. Помимо этого:

  • Функции ввода/вывода, функции для работы со строками, файлами, каталогами, графикой
  • Объектно-ориентированное программирование
    • Наследование
    • Инкапсуляция: открытые/зыкрытые/защищенные члены класса
    • Полиморфизм: перегрузка операторов/функций, виртуальные функции
    • Статические члены класса
  • Выделение памяти в куче/стеке, работа с памятью (new, delete), пространства имен, создание собственных операторов
  • Применение ассемблерных вставок
  • Функции-шаблоны, исключения
  • Стандартная библиотека шаблонов (STL)
  • Win32 API
  • МFC для построения графических windows-приложений
  • Многопоточное программирование: блокировки, мьютексы, семафоры и тому подобные ништяки.
  • Базовые моменты COM-технологии

Java

  • Базовые блоки: типы, операторы и т.п.
  • Темы, связанные с объектно-ориентированным программированием во многом аналогичны рассматриваемым на C++
  • Графические библиотеки
  • Основы технологии CORBA для создания распределенных приложений
  • Применение библиотечных классов для многопоточного программирования

C#

Здесь познания довольно обширные. По сути, это весь набор технологий в рамках средств .net: стандартные средства языка, библиотеки классов Windows Forms, WPF, ADO.net, LINQ. Также базовые знания WCF и Web Services, ASP.net. Ну и плюс опыт 1,5 года в написании клиент-серверных приложений (по большей части - клиентских).

Немного о скриптовых языках

Что касается скриптовых языков, то это Perl, Python, Ruby, ну и, пожалуй, PHP и клиентский Javascript.

Perl, Python использовались при написании заданий для дисциплины "Сетевые технологии". Сюда можно включить написание программ-клиентов и программ-серверов, работающих по основным прикладным протоколам: HTTP-клиент/сервер, SMTP, POP3, IMAP плюс утилиты trace route, ping и т.д.Данные языки также изучались самостоятельно в рамках выполнения других задач (парсинг, работа с регулярными выражениями на Perl). Ruby изучал самостоятельно: базовые элементы (вплоть до применения языка для сетевых приложений).

Javascript (в т.ч. фрэймворк jquery) также изучал самостоятельно. Использовал при написании одной курсовой (система тестирования).

PHP изучен в рамках факультатива. Применял его в серверной части веб-сайтов (в связке с Apache + MySQL/PostgreSQL).

Структуры данных

Все структуры данных изучались в рамках вышеперечисленных языков.

Наряду со стандартными для любого языка структурами (массивы, примитивные типы), стоит отметить следующие:

  • Списки (связанные/несвязанные), кортежи
  • Словари (хэш-коллекции)
  • Стэк
  • Структуры (C, C++, java) и классы (в Perl и JS реализуются как расширение функций)
  • Реляционные таблицы
  • Бинарные деревья
  • Ну и, пожалуй, структуры, предоставляющие более высокой уровень абстракции (основанные на базовых). По сути, это классы, но приобретающие в контексте использования языковых средств роль контейнеров данных. Например, DataSet в билиотеке классов ADO.net.

Алгоритмы

Все алгоритмы изучались в рамках учебных дисциплин, частично - самостоятельно. Применение алгоритмов на практике заключалось в решении задач на практикумах различных дисциплин, самостоятельной реализации некоторых алгоритмов, участии в онлайн-турнирах и отборочных турах олимпиад.

Прежде всего, это алгоритмы сортировки (пузырьком, вставками, быстрая, методом Шелла, пирамидальная). Реализовывались на C, C++, Java.

Далее алгоритмы поиска (линейный, поиск в отсортированном массиве, поиск в графе в ширину/глубину, поиск в b-дереве), рекурсия, криптографические алгоритмы (RSA и т.п.). Реализовывались на C, C++, Perl.

Парсинг в основном применял на Perl (учитывая, что язык содержит массу средств для выполнения операций такого рода).

Множество алгоритмов в таких enterprise - языках, как Java и C#, реализованы как методы библиотечных классов, предоставляя более высокий уровень абстракции, позволяя, не вдаваясь в детали реализации, сосредоточиться на общей логике программы. Исходя из этого, реализация алгоритмов для структур данных заключается в вызове вида sort([список параметров]) и говорить об изучении стандартных алгоритмов здесь неуместно.

Парадигмы

События

С данной парадигмой плотно познакомился при написании настольных приложений на C (Win32 API), C++ (MFC), C# (Windows Forms), java. Можно также сюда приписать Perl (DAEMONs) и Qt PyQt). На C# Windows Forms объем полученных знаний довольно большой (написание клиентских приложений производственного уровня).

ООП

Помимо изучения в рамках учебных дисциплин, спецкурсов, с объектно-ориентированным программированием имею дело каждый день, в т.ч. проектирование иерархии классов-сущностей, реализующих бизнес-логику в системе.

Параллельность/асинхронность

В рамках данной парадигмы можно упомянуть написание клиентских приложений (практикумы, курсовые, дипломная работа) на C/C++/C#/Java. Далее следует Perl (fork, pipe()).

Асинхронность задействовалась в таких задачах, как применение ajax-запросов с помощью jquery, использование ajax-контролов в ASP.net, ну и, само собой, в desktop-приложениях на C/C++/C#/Java.

Кэширование

По большей части это использование модулей Perl, реализующих кэширование: в памяти текущего процесса (Cache::Memory), на диске (Cache::File, CGI:Cache). Также применялись встроенные средства кэширования вывода/источников данных для решения задач на ASP.net.
